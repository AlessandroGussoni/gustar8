Things that are floating into my brain right now

Thinking about NN trained to predict next move in GO. Most of them (like good and old AlphaGo) use Conv based architecture, pipeline is something like this:

  • take a board state and the position of the next stone to be placed
  • encode board state into a numerical representation of shape (B, C, 19, 19)
  • train a model to learn a F(encoding) = next move to play

    • Basically board states are encoded in the same data format of image, with each plane c mapping different informations about stones (# of liberties, ko, etc), in order to make 2D conv works. But like why we do not encode states as graph? Each intersection would be a node and lines would be edges for an empty board. Main idea is graph semantically capture more of the true nature of the Go game, (territories, stone connections etc) so that group of stones shares same connections; then map feature planes to node features and you can train a simple GCN on that. Not sure if it makes too much sense need to think about it more
